# GIF 动图 LLM 增强插件 (astrbot_plugin_read_gif)

将发送给 LLM 的 GIF 动图自动转为宫格静态图，让多模态模型真正读懂动图内容，而非只看第一帧。

## 功能简介

当你在聊天中发送或引用 GIF 动图时，在 Astrbot 收到 GIF 文件后，绝大多数多模态模型**实际上只能看到第一帧**，导致对动图内容的理解严重不全。

本插件在后台无感拦截 GIF，将其**均匀取帧并拼成一张宫格静态图**（如 4/9/16/25 宫格），再替换原始 GIF 发给 LLM。这样 LLM 就能一眼看到动图的完整变化过程。

## 核心特性

- **全平台兼容**：Windows、Linux、Docker 均可运行，不依赖任何外部二进制
- **全平台无感替换**：支持 Telegram、QQ、Discord、Kook 等所有 AstrBot 平台，只要平台能收到 GIF，本插件就能处理
- **跨插件零冲突**：只干一件事——替换发给 LLM 的 GIF，不影响任何其他插件功能
- **智能宫格**：支持 4/9/16/25 宫格，以及"自动"模式（根据 GIF 时长智能匹配最合适的宫格数）
- **内容哈希去重**：同一张 GIF 多次发送不会重复生成宫格图，直接复用缓存
- **可观测日志**：每次处理都会输出简洁的 info 日志，方便追踪插件工作状态
- **缓存管理**：自动/手动清理缓存，支持通过命令查看和发送缓存的宫格图

## 安装

通过 Astrbot WebUI 安装本插件。

## 配置说明

在 AstrBot WebUI 的插件配置页面中，可调整以下选项：

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| 宫格预设 | 下拉 | 自动 | 可选：自动 / 4 / 9 / 16 / 25。自动模式会根据 GIF 时长智能选择 |
| 宫格图长边限制 | 整数 | 1600 像素 | 宫格输出图的最大长边分辨率，超出会自动等比缩放 |
| 自动清理缓存间隔 | 整数 | 60 分钟 | 0 表示关闭自动清理 |
| 缓存文件最大保留时长 | 整数 | 24 小时 | 超过此时长的缓存文件会被自动清理删除 |
| GIF 动图提示词 | 文本 | （见下方） | 处理 GIF 后注入给 LLM 的提示词，引导模型理解宫格图代表一段动画。留空则不注入 |
| 在 Info 日志中显示 Debug 信息 | 开关 | 关闭 | 开启后本插件所有 Debug 诊断日志会升级为 Info 级别，便于排查问题 |

### 自动宫格规则

| GIF 时长 | 自动选择的宫格数 |
|---|---|
| 0 ~ 3 秒 | 4 宫格 |
| 3 ~ 10 秒 | 9 宫格 |
| 10 ~ 20 秒 | 16 宫格 |
| 20 秒以上 | 25 宫格 |

> 如果 GIF 帧数不足所选宫格数，会自动降级到不超过帧数的最大完全平方数。

### GIF 动图提示词

处理 GIF 后，插件会用一段提示词引导模型理解"宫格图代表一段动画"。默认提示词如下，可在 WebUI 中自定义或清空（清空则不注入任何提示词）：

> 图片中可能包含宫格图。宫格图是 GIF 动图的关键帧拆分，这些帧按时间顺序排列，共同描述了一段连续画面。请把它们当作动态 GIF 来理解，禁止提及“宫格”、“拼接”等技术细节。

提示词会**按主 LLM 是否支持图片模态自动分流**到正确的位置，无需手动配置：

- **主 LLM 支持图片模态**（直接看宫格图）：提示词注入主 LLM 的 `system_prompt`。
- **主 LLM 不支持图片模态 + 配置了图片转述模型**：图片由转述模型先转成文字描述再给主 LLM。此时若把提示词加到主 LLM，会造成污染（主 LLM 看不到图，只能看到转述文字）。插件选择在转述发生时，把提示词追加到**转述模型**的请求末尾，让转述模型直接按动图来描述，而非原样描述。该机制同时覆盖**直接发送的 GIF**和**引用消息里的 GIF**两种情境，且对任意图片转述模型都生效。

## 命令

本插件注册以下命令（均可在任意会话中使用）：

### `/gifcache`

查看当前缓存的宫格图列表，或发送指定缓存图。

```
/gifcache           # 列出最近 20 个缓存文件
/gifcache 3         # 发送列表中第 3 个缓存图
/gifcache abc       # 按文件名前缀查找并发送
```

### `/gifclean`

立即清空缓存目录中的**所有**宫格图。

```
/gifclean           # 清空全部缓存文件，无论新旧
```

> 手动命令与自动清理的语义不同：
> - 手动 `/gifclean`：清空全部缓存，立即生效
> - 自动清理：仅按 WebUI 中「缓存文件最大保留时长（小时）」配置删除过期文件

## 日志示例

当插件处理一张 GIF 时，你会在 AstrBot 日志中看到类似如下信息：

```
INFO [astrbot_plugin_read_gif] GIF帧数69，秒数6.00s，已选[自动]，智能转为9宫格
INFO [astrbot_plugin_read_gif] GIF帧数80，秒数8.52s，已选[16宫格]，转为16宫格
```

当主 LLM 不支持图片模态、走图片转述模型时，日志会带提示后缀：

```
INFO [astrbot_plugin_read_gif] GIF帧数50，秒数5.00s，已选[自动]，智能转为9宫格（当前主LLM不支持图片模态，已对识图模型附加提示词）
```

一条消息里有多张 GIF 时，会附带本轮数量：

```
INFO [astrbot_plugin_read_gif] GIF帧数50，秒数5.00s，已选[自动]，智能转为9宫格，本轮共3张GIF
```

## 技术细节

- **依赖**：仅依赖 Pillow（PIL），AstrBot 环境通常已自带
- **缓存目录**：`data/plugin_data/astrbot_plugin_read_gif/`（遵循 AstrBot 插件数据目录规范）
- **输出格式**：PNG，带透明通道合成（白色背景）
- **输出尺寸限制**：宫格图长边默认最大 1600 像素（可在配置中自定义），自动等比缩放
- **处理时机**：主入口在 `OnWaitingLLMRequestEvent` 钩子中处理（早于 `build_main_agent`，避免 GIF 被压缩成单帧 JPEG）；GIF 解码与拼接通过 `asyncio.to_thread` 在线程池执行，不阻塞事件循环
- **缓存去重**：以原始 GIF 文件内容的 SHA-256 哈希（前 16 位）+ 宫格数作为缓存文件名，同一张 GIF 多次发送直接复用缓存，跳过重复解码
- **提示词注入（按主 LLM 是否支持图片模态分流）**：
  - **主 LLM 直接看图**：提示词注入 `req.system_prompt`（独立 system message，与用户文本分离），用 `[GIF_HINT_START]`/`[GIF_HINT_END]` 标记对包裹，注入前先清理旧痕迹。`system_prompt` 每轮由 `build_main_agent` 重建，正常不会跨轮残留；标记清理是防御性措施。
  - **主 LLM 不支持图片 + 配了图片转述模型**：框架对「直接发送的图片」走 `_request_img_caption`、对「引用消息里的图片」走 `_process_quote_message`（后者 prompt 为硬编码，不读 `image_caption_prompt`）。为同时覆盖这两条路径且不依赖框架的 prompt 字面量，插件在命中转述路径时**临时包装转述模型实例的 `text_chat`**：在框架真正发请求前，把提示词追加到 prompt 结尾，让转述模型按「动态序列」而非「宫格」描述。包装对任意 Provider 子类通用，本轮结束即卸载（`on_waiting` 入口、`on_llm_request`、`terminate` 三处兜底）。
  - **框架限制（如实标注）**：引用消息里若有多张图，框架只转述其中第一张，后续图不转述，这非插件可补。

## 注意事项

1. 本插件**只处理 GIF 文件**（仅通过文件头 magic bytes `GIF87a`/`GIF89a` 判断，不检查后缀。这是因为平台适配器下载的临时文件可能无后缀或使用任意后缀），PNG、JPEG、WebP 等静态图不受影响
2. 如果 GIF 只有 1 帧（伪动图），插件会自动跳过，不做处理
3. 缓存文件以 PNG 格式保存，命名包含原始 GIF 内容哈希与宫格数，同一张 GIF 重复发送会命中缓存
4. 插件对原始消息链的修改是**内存中临时替换**，不会修改聊天记录或持久化数据
5. **本插件仅对 AstrBot 内置 Agent（`agent_runner_type = local`）生效**。当配置为第三方 Agent Runner（Dify / Coze / Dashscope / DeerFlow）时，`OnWaitingLLMRequestEvent` 不会被触发，GIF 替换不生效，GIF 会以原始文件形式发给外部平台（外部平台通常只取第一帧）。插件检测到此情况会在日志中输出一条 info 提示，不会静默失效。

## 文件结构

以下展示插件本体的目录结构，以及插件在 AstrBot 运行时涉及的配置与数据目录（相对路径）。

```text
插件本体（data/plugins/astrbot_plugin_read_gif/）
├─ main.py                                   # 插件入口：Hook 注册、命令处理、缓存管理
├─ gif_processor.py                          # GIF 处理核心：帧提取、宫格拼接、自动宫格选择、内容哈希去重
├─ metadata.yaml                             # 插件元数据（名称、版本、作者等）
├─ _conf_schema.json                         # WebUI 配置定义（开关、下拉、数值、文本）
└─ README.md                                 # 本文件

AstrBot 插件配置目录（data/config/）
└─ astrbot_plugin_read_gif_config.json       # WebUI 保存的插件配置，如宫格预设、提示词等

AstrBot 插件独立数据目录（data/plugin_data/astrbot_plugin_read_gif/）
└─ gif_grid_{sha256前16位}_{宫格数}.png      # 生成的宫格缓存图（同一张 GIF 命名固定，可复用）
```
